Variables¶

Package: databricks.bundles.variables

Decorators for declaring resources accept variables as parameters alongside with their values. It allows changing resources depending on which environment they are deployed to.

Example:

from databricks.bundles.jobs import sql_file_task
from databricks.bundles.variables import variables, Variable

@variables
class MyVariables:
  warehouse_id: Variable[str]

@sql_file_task(
    file="files/my_query.sql",
    warehouse_id=MyVariables.warehouse_id,
)
def my_query_task():
    pass

It also possible to access bundle variable values within job_mutator and resource_generator decorators.

Example:

@variables
class MyVariables:
  deploy_qa_job: Variable[bool]

@resource_generator
def my_query_task_resource_generator(resource):
    @job
    def my_qa_job():
        ... # add tasks here

    if MyVariables.deploy_qa_job.get():
        yield my_qa_job

See: Databricks Asset Bundles configuration

Decorators¶

@variables(
cls: type[T],
) → type[T]¶

A decorator that initializes each annotated attribute in a class with Variable type. Variables are initialized with a path that corresponds to the attribute name. Variables should specify their type, or else they will be treated as Any. Complex types like data classes, lists or dictionaries are supported.

For example, if your databricks.yml file contains:

variables:
  warehouse_id:
    description: Warehouse ID for SQL tasks
    default: ...

You can define a class with a warehouse_id attribute:

@variables
class MyVariables:
  warehouse_id: Variable[str] # ${var.warehouse_id}

And later use it in your code as MyVariables.warehouse_id.

For accessing bundle variable values, see Variable.get().

Functions¶

variables.resolve_variable(
value: T,
) → T¶

Convenience method to resolve variable values. If the value is a variable, it will be resolved using Variable.get and returned. Otherwise, the value will be returned as is.

Data types¶

class Variable¶

Reference to a bundle variable.

Instances should be created using @variables decorator.

See: Databricks Asset Bundles configuration

path: str¶

Path to the variable, e.g. “var.my_variable”.

type: Type[T] | Any¶

Type of the variable or Any if type is unknown.

get() → T¶

Returns the value of the variable. This method can only be called within @job, @resource_generator or @job_mutator.

You can only get variables starting with “var.”.

property value: str¶

Returns the variable path in the format of “${path}”

static from_path(
path: str,
) → Variable¶

Create a variable from a given path.

Example:

Variable.from_path("var.my_variable")
