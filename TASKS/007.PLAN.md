## Plan: State migration system for permissions

### Problem
The current state format on main uses:
- `"permissions"` as JSON key (from `PermissionsState.Permissions` field)
- `iam.AccessControlRequest` with `"permission_level"` field

Our branch changed this to:
- `"_"` as JSON key (from `EmbeddedSlice` convention)
- `StatePermission` with `"level"` field

The current `UnmarshalJSON` on `PermissionsState` handles `permission_level` → `level` migration, but assumes the JSON key is already `"_"`. Real old state files from main use `"permissions"` as the key, which `json:"_"` won't match.

### Approach

1. **Create `bundle/direct/dstate/migrate.go`** with migration infrastructure:
   - Called right after state is loaded in `Open()`
   - Checks `state_version` — if < current, runs migration hooks
   - Bump `currentStateVersion` to 2
   - Migration operates on `json.RawMessage` entries in `Database.State`

2. **Permissions migration hook (version 1 → 2)**:
   - Copy old struct from main directly into `migrate.go`:
     ```go
     type oldPermissionsState struct {
         ObjectID    string                     `json:"object_id"`
         Permissions []oldPermission            `json:"permissions,omitempty"`
     }
     type oldPermission struct {
         PermissionLevel string `json:"permission_level,omitempty"`
         UserName        string `json:"user_name,omitempty"`
         ServicePrincipalName string `json:"service_principal_name,omitempty"`
         GroupName       string `json:"group_name,omitempty"`
     }
     ```
   - For each key matching `*.permissions` in state:
     - Unmarshal `json.RawMessage` into old struct
     - Convert to new `PermissionsState` (with `"_"` and `"level"`)
     - Re-marshal back to `json.RawMessage`
   - Update `state_version` to 2

3. **Clean up current migration code**:
   - Remove `UnmarshalJSON` from `PermissionsState`
   - Remove `migratePermissionLevel` function
   - Remove `PermissionLevel` field from `StatePermission` (no longer needed after migration at state level)

4. **Update acceptance test**:
   - Modify existing `permission_level_migration` test to use truly old state format with `"permissions"` key
   - Verify migration happens at load time

5. **Validate**: `make checks fmt lint`, `make generate`, `make test-unit && make test-update`
